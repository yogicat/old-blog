IP and Bytes and Human..

Feb 23, 2018 00:00 · 1432 words · 3 minute read til learning internet

IP주소와 바이트와 …

순수하게 나의 궁금증 해결을 위한 쉬는시간 틈새 공부. 네트워크 공부하다가 마주친 IP주소에서 시작된 잡포스팅.

IP주소는 Internet Protocol의 약자로, 인터넷 통신을 위한 주소이다. 인터넷은 컴퓨터들의 네트워크로 인터넷에 연결된 각각의 컴퓨터는 고유의 주소를 반드시 갖는다. 따라서 인터넷에 연결된 각각의 컴퓨터, 프린터, 스위치, 라우터, 이외의 각종 기기는 이러한 숫자로 된 주소를 갖고있다.

이 각각의 연결된 기계들은 네트워크에서는 노드라고 부른다. 네트워트 노드들은 연결되는 순간, DHCP (Dynamic Host Configuration Protocol)서버로 부터 IP주소를 할당받는다. (집주소와 같이) 이 고유 주소는 nnn.nnn.nnn.nnn의 형태로 이루어져 있고 (IPv4의 경우), 각각의 nnn은 1Byte로 0부터 255사이의 수로 나타낸다.


왜 0부터 255일까?

이를 이해하기 위해서 컴퓨터를 이루는 최소 단위인 bit 그리고 Byte에 대해 먼저 알아보자. bit는 컴퓨터 프로그래밍에서 가장 작은 저장단위로 0 또는 1을 나타낼 수 있다. 그리고 이런 bit들이 8개가 모이면 1Byte 라고 한다.(0 1 0 1 1 0 1 0) 그리고 1byte는 한개의 캐릭터(A, x, $)를 나타낼 수 있다.

1개의 bit는 0또는1을 나타내므로 2가지 값을 나타낼 수 있고, 2-bit는 4개(2 * 2), 3-bit는 8개(2 * 2 * 2), 4-bit는 2 * 2 * 2 * 2 인 16개..이렇게 2의 배수로 늘어나게 된다. 따라서 8-bit는 2의 8제곱이 되고 이는 0부터 255까지의 256개의 값을 나타낼 수 있다.

8개의 비트를 1바이트라고 표현하는데 이는 하드웨어에 의한 프로그램에 따라서 바이트가 표현하는 수는 달라질 수 있다고 한다.


왜 Byte인가?

바이트는 50년대 후반 IBM에서 만들어져서 초반에는 특정 데이터 패스의 비트수를 표현하기 위해 사용되었지만, 1960년대 중반쯤 중형 컴퓨터 시스템의 개발과정에서 8비트를 묶어서 부르는 이름으로 굳어지게 되었다. (초기에는 다양한 비트들이 사용되었음) 8비트로 나타낼 수 있는 1바이트는 ( 0 0 0 0 0 0 0 0 )부터 ( 1 1 1 1 1 1 1 1 )까지의 값을 가질 수 있다. 이 값은 0부터 255까지의 양수를 표현할 수 있다. 바이트는 세계 대부분의 문자를 저장하는 데 아주 이상적인 크기이다.(한국,중국,일본어는 2바이트를 이용하여 표현한다) 또한 바이트는 흑백 사진에서 회색조를 표현하는 단계로 사용하기에도 이상적인데, 사람의 눈이 대략 256단계 정도의 명암을 구분할 수 있기 때문이다.(CODE,Charles Petzole)

비트나 바이트나 어쨌든 결국 인간에 의해서 무언가를 전달하고 전달 받기 위해서 고안된 방법인 것 같다. 그런 면에서 재미있는 것은, 인간은 지금 시대든, 과거든, 언제든 할 것 없이, 어떻게 해서든 소통(communication)하고 싶어 했다는 점이다. 고구려 시대에도 조선시대에도 우리는 육신이 떨어져 있어도 편지나 전보를 통해 소통하길 원했고, 왕좌의 게임이나 해리포터에서도 까마귀나 부엉이를 통해서 소통하고자 했으며, 언젠가부터는 기계를 통해 소통하기 위해 다양한 프로그램 언어들이 생겨났으며, 이제는 인간이 만든 기계인 AI와 소통하려고 노력하니 정말로 인간이란 참으로 끈기 있고 호기심 많고 포기를 모르는, 사회적 동물이 맞는구나 라는걸 기술을 공부하면서 더 많이 느끼게 되고 이런 어마어마한 기술을 만들어낸 사람들과 같은 종의 생물체라니 왠지 나도 할 수 있을 거 같은 알 수 없는 힘에의 의지 같은 것이 생긴다.

IP가 궁금해서 시작한 서치가 비트와 바이트로 흘러가 인간에 대한 경이로움으로 마무리…언제쯤 제대로 정리하고 마무리 지을 수 있으려나..나도 화이팅..!

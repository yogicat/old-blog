On writing well…

코딩의 세계는 그동안 내가 거쳐온 세계들 중에서 discipline 이 가장 중요시되는 세계인 거 같다. 그리고 그렇기 때문에 참 매력적인 것 같다. 어제 javascript 역사 관련된 글을 읽다가 마주친 harmony라는 단어가 자꾸만 머릿속에서 맴돈다. 아름다운 세상을 위한 하모니는 꼭 아니지만 서로의 이익을 위해서든, 버그를 줄이기 위해서든, 어쨌든 이 컴퓨터 프로그래밍 세상에서는 모두가 늘 조화를 이루기 위해 끊임없이 노력한다는 인상을 받았다. 각양각색으로 흩뿌려진 상태에서 차근차근 하나로 모아지려는 현상이 일반 사람인 나에게는 늘 새롭다. 디자인의 세상에서도 나도 모르게 지키고 있던 모두가 생각하는 불변의? 법칙들 이 있는데 너무나 당연해서 이렇게 되기까지 얼마나 많은 노력들과 시행착오와 의견들이 있었을지 잊고 있었다. 코딩 스타일과 컨벤션을 배우면서… 새삼 오래도록 전해지는 그런 어떤 규칙들, 컨벤션들이 대단하게 느껴졌다.

what is coding convention?

코딩컨벤션이란 프로그래밍 언어를 위한 가이드 라인이라고 할 수 있다. 프로그래밍 세상에서 원작자에 의해 계속 유지되는 코드는 거의 없다. 대부분의 프로그램은 수많은 사람들의 손을 거친다. 따라서 가독성과 유지보수 관점에서 코딩컨벤션을 가지고 따르는것이 중요하다.

특히나 처음 배우는 단계에서는 이런 부분을 아직은 필요 없다고 느끼고 무시하기 쉬운데 작은 부분을 어떻게 하는지에 따라 전체가 달려있다고 생각한다. 덤벙대고 무심하고 대충대충을 좋아하는 나에게 특히나 어렵게 느껴지는 부분이라서 더욱 중요하게 생각하고 노력해야겠다.

set of guidelines for a specific programming language that recommend programming style, practices, and methods for each aspect of a program written in that language. Coding conventions - Wikipedia

Douglas Crockfords says…

 frontend master에서 더글라스 아저씨가 강조한 writing style 관련해 새겨두면 좋을 내용들

  • Programs must communicate clearly to people.
  • Confusion must be avoided.
  • Always use ===
  • Make your programs look like what they do.
  • Write in the language you are writing in.
  • ++ occurs ambiguity. use x += 1 instead.
  • Clean code is easier to reason about.
